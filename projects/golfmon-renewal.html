레저플러스서울

골프몬 리뉴얼 안정화 및 운영 반영

외부 개발 산출물을 인수해 기존 골프몬 서비스의 운영 정책, 데이터 흐름, 예외 케이스를 분석하고 누락 기능과 오류를 보완해 리뉴얼 서비스를 실제 운영 가능한 상태로 안정화한 프로젝트입니다.

Java Spring Boot PHP MySQL AWS
Role
백엔드 개발 및 운영
Focus
산출물 인수, 정책 반영, 오류 수정, 운영 안정화
Domain
골프/레저 예약 플랫폼

문제

리뉴얼 버전은 기존 골프몬 서비스와 동일하게 운영되어야 했지만, 실제 운영 정책, 화면 동작, 데이터 처리 방식 중 일부가 누락되거나 다르게 구현된 부분이 있었습니다.

운영 중인 서비스였기 때문에 단순 오류 수정뿐 아니라 예약 흐름, 관리자 업무 흐름, 기존 데이터와의 정합성까지 함께 확인해야 했습니다.

기여

  • 외부 개발 산출물의 백엔드 구조와 동작 흐름 분석
  • 기존 골프몬 서비스에 적용되어 있던 정책과 기능 확인
  • 리뉴얼 버전에 누락된 기능과 운영 중 발견된 오류 보완
  • 기존 데이터 흐름과 신규 리뉴얼 기능 사이의 정합성 검토
  • 배포 이후 운영 이슈 추적과 반복 안정화

접근 방식

  1. 기존 서비스와 리뉴얼 버전의 기능 차이를 비교했습니다.
  2. 오류가 발생하는 화면, API, 데이터 처리 구간을 확인했습니다.
  3. 기존 운영 정책이 반영되지 않은 기능을 찾아 수정 범위를 정리했습니다.
  4. 백엔드 로직, 데이터 조회/저장 흐름, 예외 처리를 보완했습니다.
  5. 운영 중 발견된 이슈를 기준으로 반복적으로 수정하고 안정화했습니다.

결과

리뉴얼 산출물을 기존 골프몬 운영 정책에 맞게 보완했고, 누락 기능과 오류를 수정해 운영 가능한 수준으로 안정화했습니다. 이 과정에서 기존 서비스의 업무 흐름과 리뉴얼 서비스의 동작 차이를 줄였습니다.
