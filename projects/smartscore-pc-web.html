PC WEB 골프장/부킹 서비스 개발

앱 중심으로 제공되던 골프장/부킹 서비스를 PC 웹 사용자가 탐색할 수 있는 흐름으로 확장했습니다. 골프장 목록, 검색/필터, Kakao 지도 이동이 끊기지 않도록 화면 상태와 백엔드 데이터를 연결하고, 반복되는 UI는 공통 컴포넌트로 정리했습니다.

PHP Vue.js MySQL JavaScript Kakao Map API
Expansion 앱 서비스를 PC 웹으로 확장

모바일 중심의 골프장/부킹 경험을 PC 화면 흐름에 맞게 재구성했습니다.

Interaction 목록 선택과 지도 이동 연결

칩 형태의 골프장 목록 선택이 Kakao 지도 위치 이동으로 이어지도록 구현했습니다.

Reuse 검색/필터 UI 공통화

반복되는 조건 입력과 필터 구조를 컴포넌트화해 화면 간 일관성을 높였습니다.

Period
2020.09 - 2021.02

PC 웹 골프장 영역 개발, 공통 컴포넌트 개발, 백엔드 연계
Focus
골프장 화면, 검색/필터, 지도 API, 공통 UI, 데이터 연계

문제

기존 서비스는 앱 중심으로 운영되고 있었기 때문에 PC 웹에서는 화면 밀도, 검색 흐름, 지도 기반 탐색 방식을 새로 정리해야 했습니다. 골프장 목록, 조건 필터, 지도 위치가 서로 따로 움직이면 사용자는 현재 보고 있는 골프장과 지도 위치를 다시 확인해야 했습니다.

대표 구현 사례

Kakao 지도와 골프장 목록 연동

골프장 목록을 칩 UI로 제공하고, 사용자가 항목을 선택하면 해당 골프장 위치로 지도가 이동하도록 연결했습니다. 화면 상태, 목록 데이터, 지도 API 호출 타이밍이 어긋나지 않도록 선택 상태와 좌표 데이터를 함께 확인했습니다.

  • 백엔드에서 내려오는 골프장 데이터를 PC 웹 목록과 지도 표시 흐름에 맞게 사용했습니다.
  • 목록 탐색과 지도 위치 확인이 하나의 동작처럼 이어지도록 인터랙션을 정리했습니다.

검색/필터 UI 공통 컴포넌트화

골프장과 부킹 화면에서 반복되는 조건 입력, 필터, 목록 갱신 흐름을 공통 컴포넌트 기준으로 정리했습니다. 같은 조건 UI가 화면마다 다르게 구현되지 않도록 구조를 맞추고, 이슈 단위로 화면 동작과 데이터 표시를 함께 검증했습니다.

  • 반복되는 검색 조건과 필터 UI의 구현 부담을 줄이고 화면 간 일관성을 높였습니다.
  • Vue.js 기반 화면 구조와 PHP 백엔드 데이터 연계 흐름을 함께 확인했습니다.

담당 범위

  • 앱 중심으로 제공되던 기능을 PC 웹 사용자가 이해할 수 있는 화면 흐름으로 재구성했습니다.
  • 골프장 관련 화면의 목록, 검색, 필터, 지도 탐색 흐름을 중심으로 작업했습니다.
  • 공통 레이아웃과 반복 사용 영역을 기준으로 Vue.js 컴포넌트 구조를 잡았습니다.
  • 목록, 상세, 검색 조건, 지도 위치처럼 사용자 화면에 필요한 데이터를 기존 백엔드 흐름과 연결했습니다.
  • Redmine, Git 기반으로 이슈 단위 작업과 협업을 진행했습니다.

결과

앱에서 제공하던 핵심 서비스를 PC 웹으로 확장하면서 사용자 접점을 넓혔고, 골프장과 부킹 서비스의 공통 웹 영역을 구성했습니다. 골프장 목록, 검색/필터, 지도 이동이 연결된 탐색 경험을 구현했고, 기존 서비스 데이터 흐름을 이해한 상태에서 웹 화면과 백엔드 연계를 함께 다뤘습니다.
